How to install the component

The components have been tested using Litmus and are compliant with the specified requirements. The code is compatible with major ESPs, including Salesforce, Braze and Acoustic, and is therefore ready for integration; for correct usage, it is recommended to follow the instructions provided in the attached ZIP file.

There may be cases in which the HTML code of your email prevents the component from functioning correctly. This is due to the presence of CSS code that interferes with its functionality.
Not a coding wizard? No problem — just ask the Email Team for a hand!

For clients using the Merkle CMS, additional adjustments are required to ensure the correct functioning of the components within the CMS environment.

The following video illustrates how to integrate the components into your HTML code and the modifications required to ensure they function correctly.

For illustrative purposes, a folder has been created containing the HTML file, along with a folder called "images" containing the images to be used, as well as the extracted ZIP folder of the component. In this case, the component used is the "3 Carousel Slide with CSS effects".

(The selected client template is for illustrative purposes only.)

Step 1:
Create a folder that contains your HTML template, a separate folder for the assets needed for the component, and the extracted ZIP folder of the interactive component itself.

Step 2:
Open your HTML template in the editor of your choice. In this tutorial, we will replace the hero image with the interactive component.

Step 3:
Back in the HTML template, identify the hero image section and delete the code, making sure the rest of the markup still includes all necessary closing tr, td, and table tags.

Step 4:
Read the instructions in the README file carefully.
Copy and paste the CSS code of the interactive component as indicated.

Step 5:
Read the instructions in the README file carefully.
Copy and paste the HTML code of the interactive component to replace the hero section.
Preview the HTML email in a browser to view the component and check for any errors.

Step 6:
Read the instructions in the README file carefully.
Check the width of the email and make any necessary adjustments to the interactive component. In this case, the width of the email and the component match: both are 600 px, so no code changes are required.

Step 7:
We proceed with the integration of the images and the fallback image.
In the absence of a dedicated fallback asset, the first available image has been used.
After adding the images and saving the file, simply refresh the email page in the browser to verify the correct functioning of the carousel; as expected, the component is fully compatible with mobile devices
